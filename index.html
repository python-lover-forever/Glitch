🚀 Glitch the Internet

Build a live website that intentionally breaks reality in the browser.

Your site must include:

  • At least one intentional glitch (visual, interaction, or logic)
  • Be fully deployed and accessible online
  • Create a weird, surprising, or confusing experience

Examples of glitches:

  • Broken layouts / shifting UI
  • Fake system errors or terminals
  • Infinite loops / trapped interactions
  • Sound-based chaos
  • “Hacking” illusions
  • Anything that makes the user say: “what just happened?”

What Will You Learn

Students will learn Advanced Frontend Engineering + Creative Coding.

  • Deep DOM Manipulation (dynamic changes, runtime control)
  • Creative Coding (turn ideas into interactive experiences)
  • Advanced JavaScript (async, loops, user flow)
  • Browser Rendering (reflow, repaint, breaking expectations)
  • UX Psychology (confusion, surprise, tension)
  • Interactive Systems Design (game mechanics, triggers)
  • Debugging Chaos (control glitches, bug vs feature)
  • Shipping Real Projects (hosting, domains, performance)

Participants will learn how to turn the web into a creative medium—combining JavaScript engineering, browser internals, and experimental design to build interactive experiences that challenge how users perceive reality online.
